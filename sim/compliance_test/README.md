# Compliance Tests

This folder runs ACT4/Sail golden signature checks for the DUT.

Golden files are generated by an external WSL-side tool, then copied into this
CPU repo under `sim/compliance_test/`.

## What This Folder Uses

```text
sim/compliance_test/
  runner.py             # shared compliance runner
  test_one.py           # run one compliance test
  test_all.py           # run all compliance tests
  golden_sig/           # generated Sail golden signatures
  DUT_metadata/         # generated per-test metadata
  DUT_runtime/          # generated binaries/data/output
```

The generated folders come from:

- [riscv/riscv-arch-test](https://github.com/riscv/riscv-arch-test)
- [WsWSC/3-stage-riscv-golden-generator](https://github.com/WsWSC/3-stage-riscv-golden-generator)

For generator internals and troubleshooting, see the generator repo README.

## Generate Golden Files

Run these steps in WSL.

Step 1. Clone the official ACT4 repo:

```bash
mkdir -p ~/risc-v
cd ~/risc-v
git clone https://github.com/riscv/riscv-arch-test.git
```

Step 2. Clone the golden generator repo:

```bash
cd ~/risc-v
git clone https://github.com/WsWSC/3-stage-riscv-golden-generator.git
```

Step 3. Create the generator config:

```bash
cd ~/risc-v/3-stage-riscv-golden-generator
cp config.env.example config.env
```

Step 4. Edit `config.env` and point `TARGET_REPO` to this CPU repo as seen
from WSL:

```bash
TARGET_REPO=/mnt/c/Users/<windows_user>/Documents/3-stage_RISC-V
```

Step 5. Generate golden files:

```bash
cd ~/risc-v/3-stage-riscv-golden-generator
./generate_golden.sh
```

Expected output in this CPU repo:

```text
sim/compliance_test/golden_sig/*.sig
sim/compliance_test/DUT_metadata/*.json
sim/compliance_test/DUT_runtime/bin/*.bin
sim/compliance_test/DUT_runtime/data/*.data
```

Expected count: `47` files in each output group.

## Run Compliance Tests

Run these steps from Windows PowerShell at the CPU repo root.

```powershell
cd C:\Users\<windows_user>\Documents\3-stage_RISC-V
```

Run one compliance test:

```powershell
python sim\compliance_test\test_one.py add
python sim\compliance_test\test_one.py beq
```

Run all compliance tests:

```powershell
python sim\compliance_test\test_all.py
```

Do not run multiple compliance commands in parallel. These scripts share
`sim/inst_data.txt` and `sim/out.vvp`.

## Common Failures

| Message / Symptom | Fix |
| --- | --- |
| `missing compliance runtime binary` | Run the WSL golden generator. |
| Stale golden files | Regenerate golden files, then rerun the Windows test. |
| Generator setup or tool errors | See the generator repo README. |
